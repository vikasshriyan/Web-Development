About the Experiment

Node.js is an open source, cross-platform runtime environment for server-side and networking applications. Node.js applications are written in JavaScript, and can be run within the Node.js runtime on OS X, Microsoft Windows, Linux and FreeBSD. Node.js provides an event-driven architecture and a non-blocking I/O API that optimizes an application's throughput and scalability. These technologies are commonly used for real-time applications. Node.js uses the Google V8 JavaScript engine to execute code, and a large percentage of the basic modules are written in JavaScript. Node.js contains a built-in library to allow applications to act as a Web server without software such as Apache HTTP Server or IIS. You can download NodeJS from here. In this experiment, I've hosted my own server "localhost:3000". I ran the JS file on the cmd prompt which starts listening on the server. Then on the client I tried to access the server using a simple "get" function. The two images are the output for the different "/Hello"" and "/GoodBye" get functions.

Code Snippet

Express is the framework used here which is declared as required, require('express') will import the require library to run the application using Express framework.
Next we create an instance and using that instance app.get is supported http method in node.js, app.get will send response to the client . Here when the server hits the "localhost:3000" it checks for /Hello or /Goodbye, and based on that the js will send a response to the client using res.send.


Javascript Code
var express = require('express');
var app = express(); //create an instance
app.get('/Hello', function (req, res) { // get function to get the url
res.send('Hello World from NodeJS'); // send response to the client
});
app.get('/Goodbye', function (req, res) {
res.send('GoodBye from NodeJS');
});
app.listen(3000) // listen on port 3000
